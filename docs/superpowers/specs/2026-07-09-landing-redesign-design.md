# Редизайн лендинга Sheber Qonaq

## Контекст

Папка `landing/` в репозитории сейчас пустая (только `vercel.json`, деплой на Vercel уже настроен). Нужно создать с нуля современный, небанальный лендинг с двумя страницами: главная и FAQ. Реальных скриншотов приложения нет — используются стилизованные абстрактные мокапы экранов.

Существующий продукт (`frontend/`) даёт фирменную палитру и шрифт:
- Primary: голубой градиент от `#ECFBFF` (50) до `#13404F` (950), базовый акцент `#60CCED` (500), `#2A8FB0` (700)
- Шрифт: Inter
- Существующие анимации в проекте: `slide-up`, `fade-in` (CSS keyframes)

## Визуальный стиль

Светлый фон с ярким фирменным градиентом как акцентом (не тёмная тема — тестировали тёмный вариант, показался слишком "тяжёлым"). Объёмные карточки с мягкими тенями (уже есть `boxShadow.card`/`sheet` в tailwind.config.js продукта — переиспользовать те же значения теней для консистентности бренда). Плавные scroll-reveal анимации (fade + slide-up при появлении секций в вьюпорте), лёгкий параллакс или анимированный градиент в hero-секции для "живости". Не шаблонный SaaS-лендинг: избегать клише вроде generic stock-фото людей за ноутбуками — вместо этого форма, цвет, абстрактные мокапы UI и типографика делают тяжёлую работу.

## Технический подход

Чистый статический HTML/CSS/JS, без фреймворка и без сборки — соответствует тому, что `landing/` уже сконфигурирован под статику на Vercel (`cleanUrls: true` в `vercel.json`, что означает `/faq.html` будет доступен по `/faq`).

Структура файлов:
```
landing/
  index.html
  faq.html
  css/
    styles.css       — общие стили: переменные бренда, типографика, layout, компоненты
    animations.css    — keyframes и scroll-reveal классы
  js/
    main.js           — IntersectionObserver для scroll-reveal, мобильное меню
  assets/
    (SVG-мокапы экранов, инлайн или как отдельные .svg файлы)
  vercel.json          — существующий, не менять
```

Никаких npm-зависимостей. Анимации — через CSS transitions/keyframes + `IntersectionObserver` (добавляет класс `.visible` при попадании секции во вьюпорт). Без анимационных библиотек (GSAP и т.п.) — достаточно нативного CSS для плавности без лишнего веса.

## Страница 1: Главная (`index.html`)

### Header (sticky)
Лого "Sheber Qonaq" слева, справа — ссылка "FAQ" и кнопка-CTA "Написать в WhatsApp" (переход на `https://wa.me/77478048183`). При скролле — полупрозрачный фон с blur (glassmorphism-подложка), чтобы не перекрывать контент грубо.

### Hero
- Крупный заголовок (проблема/обещание, конкретный текст утвердить на этапе копирайта в плане реализации)
- Подзаголовок с сутью продукта — учёт заездов/бронирований для хостелов без хаоса
- Основной CTA: кнопка "Написать в WhatsApp" → `https://wa.me/77478048183`
- Визуально: анимированный градиентный фон (тихо движущиеся blob-формы или градиентная сетка), возможно один крупный мокап экрана (Dashboard) с лёгким плавающим эффектом (`animation: float`)

### Как это работает (3 шага)
Три карточки в ряд (на мобильном — стопкой): "Зарегистрировался" → "Добавил объект" → "Ведёшь учёт". Каждая карточка — иконка/мини-мокап + короткое описание. Scroll-reveal с последовательной задержкой (stagger) — карточки появляются одна за другой.

### Экраны приложения
Секция с 3-4 стилизованными абстрактными мокапами (не реальные скриншоты — геометрические представления интерфейса: блоки, псевдо-таблицы, псевдо-график, цвета бренда). Кандидаты на показ: Dashboard, карта загрузки (Occupancy), Финансы. Карточки с hover-эффектом (лёгкий подъём + усиление тени).

### Sheber Qonaq vs Excel/блокнот
Сравнение в формате "было / стало" — две колонки или карточки. Не называть конкретных конкурентов-CRM, только противопоставление ручному учёту в таблицах/блокноте/тетради. 4-5 пунктов сравнения (например: "Двойные брони" vs "Проверка пересечений автоматом", "Забыл посчитать долг" vs "Финансы считаются сами", и т.д. — конкретные пункты уточняются в плане реализации на основе реальных фич продукта).

### Отзывы (заглушка)
2-3 карточки-плейсхолдера с местом под будущий реальный отзыв (аватар-плейсхолдер, имя-плейсхолдер, текст-плейсхолдер). Явно не выдавать за настоящие — нейтральный дизайн, который легко заменить позже без ощущения "обмана", если останется на проде временно.

### Финальный CTA
Полноширинная секция перед футером, повтор кнопки "Написать в WhatsApp" с более настойчивым/тёплым текстом-приглашением.

### Footer
Контакты (WhatsApp номер как текст + ссылка), ссылка на FAQ, копирайт.

Вопросы и ответы — как в `faq-signup.md` (о регистрации, триале, лимитах, что после 30 дней, как оформить подписку, проблемы с письмом, несколько объектов на email). ## Что осознанно не делаем (out of scope)

- Никакой формы регистрации/лидогенерации на лендинге — весь путь конверсии идёт через WhatsApp
- Никакого отдельного блока с тарифами на главной — упоминание триала остаётся коротким в hero/FAQ, не отдельной секцией
- Не используем реальные скриншоты приложения (их нет) — только стилизованные абстрактные мокапы
- Не называем конкретных конкурентов, сравнение только с "ручным учётом"
- Не трогаем существующий `frontend/` React-проект — это полностью отдельная статика
